Purple has evolved from a codebase-specific implementation into a portable, universal AI development framework — ready to drop into any project. Two new versions are available to subscribers.

Architecture Changes

  • Portable by design — No hardcoded repos, branches, or org names. Drop .claude/ into any project.
  • purple/ folder — Replaces workbench/. Clean separation: standards, documentation, temp.
  • 3 QA agents — Web (Playwright), API (curl/jq), CLI (tmux).
  • 7 focused commands — Streamlined from 13+. Project-specific commands removed.
  • Cleaner pipeline — Autonomous end-to-end with no stops between phases.
  • Multi-repo — repos/ directory for multi-repo workspaces.
vB Agent Teams

From command-and-control orchestration to autonomous team collaboration.

Capability Before (vA) Now (vB)
Agent lifecycle Spawn → execute → return → die Spawn → claim tasks → message teammates → shutdown
Task coordination Orchestrator assigns work by phase Shared task list — engineers self-claim unblocked tasks
Communication None between agents Auto-delivered messages (completions, blockers, changes)
QA fix loop QA → orchestrator → /refine → new agent QA messages the responsible engineer directly
Parallelism Parallel within phases, sequential across ALL engineers spawned at once, work across phases
Ownership Implicit file isolation Explicit ownership zones per engineer

How It Works

  1. Team creation — Named team (impl-{feature}) with shared task list
  2. Task mapping — ALL tickets loaded upfront with dependencies
  3. Engineer spawning — 1–4 engineers, each with an ownership zone
  4. Autonomous work — Claim tasks, build, message about interface changes
  5. QA integration — QA agents join team, message engineers with failures
  6. Structured shutdown — Confirm or reject with reason

Agent Upgrades

All 4 operational agents received "When Working as a Team Member" sections:

  • How to claim and prioritize tasks from the shared list
  • When to message teammates (completions, blockers, interface changes)
  • How to handle QA failure messages and create fix tasks
  • Shutdown protocol (approve/reject with reason)
